## 第一章：项目架构总览

### 1.1 前后端分离是什么？

```
你的电脑（浏览器）          服务器
┌──────────────┐    HTTP请求    ┌──────────────┐
│  Vue3 前端    │ ───────────► │  Spring Boot  │
│  (画页面)     │ ◄─────────── │  (处理数据)    │
└──────────────┘    JSON响应    └──────┬───────┘
                                      │ SQL
                                      ▼
                                   ┌──────┐
                                   │ MySQL │
                                   │ (存数据)│
                                   └──────┘
```

- **前端（frontend/）**：用 Vue 3 写的网页，负责"长什么样"——按钮、表格、图表。运行在用户的浏览器里。
- **后端（backend/）**：用 Spring Boot 写的服务端程序，负责"逻辑"——登录验证、增删改查、权限控制。运行在服务器上。
- **数据库（MySQL）**：负责"存数据"——用户信息、社团信息、活动信息全部存在这里。

它们之间通过 **HTTP 请求** 通信，数据格式是 **JSON**。前端不认识数据库，后端不认识页面，各干各的活。

### 2.4 逻辑删除 vs 物理删除

- **物理删除**：`DELETE FROM sys_user WHERE id=1` — 数据真的没了
- **逻辑删除**：`UPDATE sys_user SET deleted=1 WHERE id=1` — 数据还在，只是标记为已删除

MyBatis-Plus 配置了逻辑删除后，你调 `deleteById()`，它自动生成的是 UPDATE 语句。查数据时也会自动加 `WHERE deleted=0`。

## 第三章：后端代码逐层拆解

### 3.1 后端分层架构

```
Controller 层（接待员）
  │ 接收 HTTP 请求，调用 Service，返回结果
  │ 不写业务逻辑
  ▼
Service 层（业务员）
  │ 写核心业务逻辑
  │ 比如：查数据库 → 判断状态 → 生成token
  │ 通过 Mapper 操作数据库
  ▼
Mapper 层（翻译员）
  │ 把 Java 对象翻译成 SQL 语句
  │ MyBatis-Plus 帮你自动生成大部分 CRUD
  ▼
Entity 层（模特）
  │ 纯粹的数据容器，和数据库表一一对应
  │ 不包含任何逻辑
```

```java
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {

        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOriginPatterns(Arrays.asList("*"));  // 允许所有来源
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
```

#### 3.3.2 安全模块（security/）

**JwtUtils.java — JWT 工具（令牌制造机）**

```java
@Component
public class JwtUtils {

    @Value("${jwt.secret}")
    private String secret;       // 密钥

    @Value("${jwt.expiration}")
    private Long expiration;     // 过期时间（毫秒）

    // 生成Token
    public String generateToken(Long userId, String username, Integer role) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("userId", userId);
        claims.put("username", username);
        claims.put("role", role);
        return Jwts.builder()
            .setClaims(claims)
            .setSubject(username)
            .setIssuedAt(new Date(System.currentTimeMillis()))
            .setExpiration(new Date(System.currentTimeMillis() + expiration))
            .signWith(SignatureAlgorithm.HS256, secret)
            .compact();
    }

    // 从Token提取userId
    public Long extractUserId(String token) {
        return extractClaim(token, claims -> claims.get("userId", Long.class));
    }

    // 验证Token是否有效
    public Boolean validateToken(String token) {
        try {
            return !isTokenExpired(token);
        } catch (Exception e) {
            return false;
        }
    }
}
```

JWT 三段式：`Header.Payload.Signature`

```
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoiYWRtaW4iLCJyb2xlIjoxfQ.xxxxx
│                    │                                                    │
Header               Payload（用户数据）                                   Signature（签名）
{"alg":"HS256"}      {"userId":1,"username":"admin","role":1}             Hash(Header+Payload+Secret)
```

**JwtAuthenticationFilter.java — 每次请求经过的"门卫"**

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired
    private JwtUtils jwtUtils;

    @Override
    protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain filterChain
    ) throws ServletException, IOException {

        // 1. 从请求头取token
        String token = getTokenFromRequest(request);

        // 2. 如果有token且有效
        if (StringUtils.hasText(token) && jwtUtils.validateToken(token)) {
            Long userId = jwtUtils.extractUserId(token);
            Integer role = jwtUtils.extractRole(token);

            // 3. 创建认证对象，放入SecurityContext
            UsernamePasswordAuthenticationToken authentication = 
                new UsernamePasswordAuthenticationToken(
                    userId,   // 后面Controller用它获取当前用户
                    null,
                    Collections.singletonList(new SimpleGrantedAuthority("ROLE_" + role))
                );
            SecurityContextHolder.getContext().setAuthentication(authentication);
        }

        // 4. 继续传递请求
        filterChain.doFilter(request, response);
    }

    private String getTokenFromRequest(HttpServletRequest request) {
        String bearerToken = request.getHeader("Authorization");
        if (StringUtils.hasText(bearerToken) && bearerToken.startsWith("Bearer ")) {
            return bearerToken.substring(7);  // 去掉 "Bearer " 前缀
        }
        return null;
    }
}
```

#### 3.3.8 Controller 层（controller/）

```java
@RestController           // 声明这是REST接口控制器
@RequestMapping("/api/club")  // 所有接口以 /api/club 开头
public class ClubController {

    @Autowired
    private ClubService clubService;

    // GET /api/club/page?current=1&size=10&keyword=篮球
    @GetMapping("/page")
    public Result<Page<ClubVO>> getClubPage(
            @RequestParam(defaultValue = "1") Integer current,
            @RequestParam(defaultValue = "10") Integer size,
            @RequestParam(required = false) String keyword,
            @RequestParam(required = false) Long typeId,
            @RequestParam(required = false) Integer status) {
        return Result.success(clubService.getClubPage(current, size, keyword, typeId, status));
    }

    // POST /api/club  body: {name: "篮球社", typeId: 3, description: "..."}
    @PostMapping
    @Log(operation = "创建社团")
    public Result<Void> createClub(@RequestBody ClubDTO dto) {
        clubService.createClub(dto);
        return Result.success();
    }

    // PUT /api/club/audit/5?status=1
    @PutMapping("/audit/{id}")
    @Log(operation = "审核社团")
    public Result<Void> auditClub(@PathVariable Long id, @RequestParam Integer status,
                                   @RequestParam(required = false) String rejectReason) {
```

```java
    public void logPointCut() {}

    @Around("logPointCut()")
    public Object around(ProceedingJoinPoint point) throws Throwable {
        long beginTime = System.currentTimeMillis();
        Object result = null;
        Exception exception = null;

        try {
            result = point.proceed();  // 执行原方法
        } catch (Exception e) {
            exception = e;
            throw e;
        } finally {
            long time = System.currentTimeMillis() - beginTime;
            saveLog(point, time, exception);  // 保存日志
        }
        return result;
    }
}
```

### 6.1 UserServiceImpl.login() — 登录核心逻辑

```java
@Override
public LoginVO login(LoginDTO dto) {
    // dto = {username: "admin", password: "666666"}

    // 第1步：从数据库查用户
    User user = userMapper.selectOne(
        new LambdaQueryWrapper<User>().eq(User::getUsername, dto.getUsername())
        // SQL: SELECT * FROM sys_user WHERE username = 'admin' AND deleted = 0
    );

    // 第2步：用户不存在 → 抛异常
    if (user == null) {
        throw new RuntimeException("用户不存在");
    }

    // 第3步：校验密码
    if (!passwordEncoder.matches(dto.getPassword(), user.getPassword())) {
        throw new RuntimeException("密码错误");
    }

    // 第4步：检查账号是否被禁用
    if (user.getStatus().equals(StatusEnum.DISABLE.getCode())) {
        throw new RuntimeException("账号已被禁用");
    }

    // 第5步：生成JWT Token
    String token = jwtUtils.generateToken(user.getId(), user.getUsername(), user.getRole());

    // 第6步：组装返回对象
    LoginVO vo = new LoginVO();
    vo.setToken(token);
    vo.setUser(convertToVO(user));  // User → UserVO（去掉password，加上roleName）
    return vo;
}
```

### 6.2 ApplyServiceImpl.auditApply() — 审核申请（核心业务）

```java
@Override
public void auditApply(Long applyId, Integer status, String rejectReason) {
    Long userId = (Long) SecurityContextHolder.getContext().getAuthentication().getPrincipal();
    User user = userMapper.selectById(userId);

    Apply apply = applyMapper.selectById(applyId);
    if (apply == null) {
        throw new RuntimeException("申请不存在");
    }

    // 权限校验：社长只能审核自己社团的申请
    if (user.getRole().equals(RoleEnum.PRESIDENT.getCode())) {
        Club club = clubMapper.selectById(apply.getClubId());
        if (club == null || !club.getPresidentId().equals(userId)) {
            throw new RuntimeException("只能审核自己社团的申请");
        }
    }

    // 状态校验：只能审核"待审核"的申请
    if (!apply.getStatus().equals(ApplyStatusEnum.PENDING.getCode())) {
        throw new RuntimeException("该申请已处理");
    }

    // 更新申请状态
    apply.setStatus(status);
    apply.setRejectReason(rejectReason);
    apply.setProcessTime(LocalDateTime.now());
    applyMapper.updateById(apply);

    // 如果审核通过，把用户加入社团成员表
    if (status.equals(ApplyStatusEnum.APPROVED.getCode())) {
        ClubMember member = new ClubMember();
        member.setClubId(apply.getClubId());
        member.setUserId(apply.getUserId());
        member.setUserName(apply.getUserName());
        member.setStatus(StatusEnum.ENABLE.getCode());
        member.setJoinTime(LocalDateTime.now());
        clubMemberMapper.insert(member);

        // 更新社团成员数 +1
        Club club = clubMapper.selectById(apply.getClubId());
        club.setMemberCount(club.getMemberCount() + 1);
        clubMapper.updateById(club);
    }
}
```

### 6.3 ActivityServiceImpl.signUpActivity() — 活动报名

```java
@Override
public void signUpActivity(Long activityId) {
    Long userId = (Long) SecurityContextHolder.getContext().getAuthentication().getPrincipal();
    User user = userMapper.selectById(userId);
    Activity activity = activityMapper.selectById(activityId);

    // 校验1：活动必须是"已发布"状态
    if (!activity.getStatus().equals(ActivityStatusEnum.PUBLISHED.getCode())) {
        throw new RuntimeException("活动未开放报名");
    }

    // 校验2：人数未满
    if (activity.getCurrentParticipants() >= activity.getMaxParticipants()) {
        throw new RuntimeException("报名人数已满");
    }

    // 校验3：报名截止时间
    if (LocalDateTime.now().isAfter(activity.getSignUpEnd())) {
        throw new RuntimeException("报名已截止");
    }

    // 校验4：必须是社团成员
    Long memberCount = clubMemberMapper.selectCount(
        new LambdaQueryWrapper<ClubMember>()
            .eq(ClubMember::getClubId, activity.getClubId())
            .eq(ClubMember::getUserId, userId)
            .eq(ClubMember::getStatus, 1)
    );
    if (memberCount == 0) {
        throw new RuntimeException("只有社团成员才能报名该活动");
    }

    // 校验5：不能重复报名
    Long count = signUpMapper.selectCount(
        new LambdaQueryWrapper<ActivitySignUp>()
            .eq(ActivitySignUp::getActivityId, activityId)
            .eq(ActivitySignUp::getUserId, userId)
    );
    if (count > 0) {
        throw new RuntimeException("已报名该活动");
    }

    // 全部通过，创建报名记录
    ActivitySignUp signUp = new ActivitySignUp();
    signUp.setActivityId(activityId);
    signUp.setUserId(userId);
    signUp.setUserName(user.getNickname());
    signUp.setStatus(ApplyStatusEnum.PENDING.getCode());
    signUp.setSignUpTime(LocalDateTime.now());
    signUp.setChecked(0);
    signUpMapper.insert(signUp);

    // 更新活动当前报名人数
    activity.setCurrentParticipants(activity.getCurrentParticipants() + 1);
    activityMapper.updateById(activity);
}
```

**5 个连续校验：**

| 校验 | 防什么 |
|------|--------|
| 活动状态=已发布 | 防止报名草稿或已结束的活动 |
| 人数未满 | 防止超额报名 |
| 报名未截止 | 防止过了截止时间还能报 |
| 必须是社团成员 | 防止外人报内部活动 |
| 不能重复报名 | 防止一个人报两次 |

## 第七章：答辩准备

### 7.1 高频问题 & 回答方向

**Q1: 为什么选前后端分离架构？**

前后端独立开发部署，前端用 Vue 生态丰富，后端 Spring Boot 成熟稳定，RESTful 接口解耦，可独立扩展。

**Q2: JWT 认证流程？**

登录成功生成 token → 存 localStorage → 请求带 Authorization header → Filter 解析验证 → SecurityContext 设置当前用户。JWT 三段式：Header.Payload.Signature，用密钥签名防篡改。

**Q3: 怎么实现角色权限控制？**

三层保障：
1. 前端显示层：Sidebar 用 v-if 控制菜单渲染
2. 前端路由层：路由守卫检查 meta.roles
3. 后端接口层：Service 层通过 SecurityContextHolder 获取角色判断

**Q4: MyBatis-Plus 比原生 MyBatis 好在哪？**

内置 CRUD、分页插件、LambdaQueryWrapper 链式查询、逻辑删除、自动填充，减少 80% 的模板代码。

**Q5: AOP 日志怎么实现的？**

自定义 `@Log` 注解 → `LogAspect` 用 `@Around` 环绕通知拦截 → 记录操作人、方法、参数、IP、耗时、是否异常 → 存入系统日志表。

**Q6: 逻辑删除和物理删除的区别？**

物理删除是 DELETE 直接删数据；逻辑删除是 UPDATE 把 deleted 改为 1，数据还在。MyBatis-Plus 配置后自动转换。
